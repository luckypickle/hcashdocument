Detailed Description

While evaluating a transaction there is a lot of intermediate state that must be tracked. Any shares withdrawn from the database must be stored in the transaction state until they are sent back to the database as either new balances or as fees collected.

Some outputs such as markets, options, etc require certain payments to be made. So payments made are tracked and compared against payments required.
