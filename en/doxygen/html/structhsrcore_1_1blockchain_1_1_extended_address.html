Detailed Description

encapsulates an encoded, checksumed public key in binary form. It can be converted to base58 for display or input purposes and can also be constructed from an ecc public key.

An valid extended_address is 20 bytes with the following form:

First 3-bits are 0, followed by bits to 3-127 of sha256(pub_key), followed a 32 bit checksum calculated as the first 32 bits of the sha256 of the first 128 bits of the extended_address.

The result of this is an extended_address that can be 'verified' by looking at the first character (base58) and has a built-in checksum to prevent mixing up characters.

It is stored as 20 bytes.
